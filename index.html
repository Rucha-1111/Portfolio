Profile

I’m a Java backend developer who enjoys building REST APIs and working with databases. I try to write code that’s clear, maintainable, and stable enough to survive sudden requirement plot twists without turning into chaos. I focus more on structure than rushing endpoints, and if something looks unnecessarily complicated, I usually try to simplify it before it haunts me later.


I focus on designing REST APIs with a request–response flow that won’t make future-me cry, structuring data models and database interactions so they actually make sense later, and improving reliability and readability instead of just “making it work.” I make it a point to break down requirements before touching code, mapping business needs to backend logic that actually does what it’s supposed to, and writing documentation clear enough that future-me won’t hate past-me.


Beyond coding, I have a soft spot for business analysis and writing. I enjoy understanding problems from a business perspective and translating them into technical solutions that actually solve the right problem, while learning cloud fundamentals and experimenting with better ways to structure backend systems.
